## 🤝 Правила разработки (Конституция проекта)
1. **Правило 50/50:** Ни одна строчка кода не пишется и ни одна таблица не создается без обоюдного утверждения. Сначала логика — потом код.
2. **Качество важнее скорости:** Мы никуда не спешим. Если есть решение сложнее, но правильнее для архитектуры — выбираем его.
3. **Обязанности ИИ:** ИИ обязан критиковать идеи пользователя, если видит изъяны, и всегда предлагать лучшие альтернативы из индустрии.
4. **Обязанности Пользователя:** Пользователь имеет финальное слово в векторе развития продукта (UX, удобство в зале, бизнес-логика).
